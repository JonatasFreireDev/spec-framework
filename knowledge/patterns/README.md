# Patterns

## Purpose

Store reusable product, UX, engineering, analytics, testing, and release patterns.

## When To Use

Use this folder when the same solution shape appears across multiple artifacts and should be applied consistently.

## Expected Files

- `README.md`: folder purpose and usage.
- Future pattern files such as `idempotent-mutations.md`, `scanner-states.md`, `release-gates.md`, or `analytics-events.md`.

## Responsible Skill

Primary owner: Documentation Orchestrator.

Supporting skills: UX/UI AI, Implementation Planner AI, QA AI, Release Orchestrator.

## Next Step

Patterns should reference examples and decisions. If a pattern changes architecture or product behavior, request approval first.
