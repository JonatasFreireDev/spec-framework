# Domains

Domains are stable business areas. Each domain owns user goals; goals own features; features own verifiable use cases.

## Start here

1. Confirm the applicable Foundation or Feature Brief gate in `../BOOTSTRAP.md`.
2. Read the pinned `examples/events/domains/events/domain.md` as the boundary reference.
3. Copy `_template-domain/` to a stable business-area slug and replace every `TBD`.
4. Define ownership, non-ownership, dependencies, and one Domain → Goal → Feature → Use Case walking skeleton.

Do not use the product name, a screen, or a catch-all label as a domain. Authentication normally belongs to an identity boundary unless an approved decision says otherwise.
