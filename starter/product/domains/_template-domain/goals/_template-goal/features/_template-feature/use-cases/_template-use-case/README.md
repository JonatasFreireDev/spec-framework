# Template Use Case

## Purpose

Copy this folder for a single verifiable product interaction.

## Expected Files

| File | Purpose |
| --- | --- |
| `context.md` | Use case identity, tier, parents, artifacts, and next skill. |
| `use-case.md` | Interaction narrative and acceptance scope. |
| `specification.md` | Central implementation contract. |
| `design.md` | UX/UI plan or explicit `Not applicable`. |
| `implementation-plan.md` | Technical plan and sequencing. |
| `execution-graph.json` | DAG of task files. |
| `tasks/` | Canonical one-file task artifacts. |
| `tasks.md` | Generated task index. |
| `tests.md` | Test strategy and acceptance evidence plan. |
| `analytics.md` | Events and measurement plan. |
| `qa-evidence.md` | Real QA evidence after implementation. |
| `security-review.md` | Security review for executable or sensitive work. |
| `code-review.md` | Read-only implementation review before validation. |
| `audit.md` | Final audit and release readiness notes. |
