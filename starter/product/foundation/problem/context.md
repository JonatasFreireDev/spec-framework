## Purpose

Use this context to keep the initial product problem grounded in evidence instead of solution ideas.

## Definition Of Ready

| Check | Expected Evidence |
| --- | --- |
| User segment named | A specific audience, not "everyone". |
| Pain described | Observable problem in the user's language. |
| Evidence captured | Interview, research, analytics, support signal, or credible assumption label. |
| Opportunity framed | Why solving this now is worthwhile. |
