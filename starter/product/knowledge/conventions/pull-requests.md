# Product Pull Request Convention

## Snapshot

| Field | Value |
| --- | --- |
| Status | `draft` |
| Owner | Product adopter |
| Purpose | Define the review surface required before validation or release. |

## PR Checklist

| Check | Evidence |
| --- | --- |
| Linked tasks | Paths to task files. |
| Specification alignment | Link to use case specification. |
| Tests | Command output or CI URL. |
| QA Evidence | Link to `qa-evidence.md`. |
| Code Review | Link to `code-review.md` or PR review. |
| Security Review | Required for Tier L or sensitive changes. |
| Rollback | Notes or runbook when applicable. |

## Rule

Do not mark tasks or use cases as `validated` without a PR or equivalent review surface.
