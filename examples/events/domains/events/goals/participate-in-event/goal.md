# User Goal: Participate In Event

## Context

- ID: GOAL-001
- Status: draft
- Domain: DOMAIN-001
- Context file: context.md

## User Intent

As an attendee, I want to participate in an event I joined, so I can have the social experience I expected and be recognized as present.

## Why This Goal Matters

Participation is closer to real product value than event browsing or RSVP intent. It connects discovery to actual offline social connection.

## Journey Summary

- Trigger: attendee decides to go to an event.
- Main path: opens event details, arrives, checks in, and is marked as present.
- Success moment: attendee is admitted or recognized as present.
- Failure moments: attendee cannot find proof, QR is invalid, network fails, organizer cannot validate.

## Candidate Features

- FT-001 - QR Code Check-in - draft

## Rules And Constraints

- Check-in should minimize personal data exposure.
- Check-in should resist duplicate or forged confirmations.

## Metrics

- Check-in completion rate.
- Event attendance conversion from RSVP.
- Check-in failure rate.

## Risks And Open Questions

- Offline behavior needs a product decision.
- QR expiration policy needs a security decision.
