# Example Journeys

## Status

Draft example. Not product scope.

## Purpose

Journeys describe how a user moves toward the goal across moments, decisions, states, and possible failures.

## Example Journey Shape

1. User enters the domain from a trigger.
2. User evaluates whether the goal is relevant.
3. User takes an action supported by a feature.
4. System confirms success or explains failure.
5. User sees the next useful step.

## Required Detail For Real Goals

- Actor and scenario.
- Entry point.
- Steps and decisions.
- Emotional or operational friction.
- States, errors, and recovery.
- Metrics and evidence.
