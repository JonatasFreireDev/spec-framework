# Example Goal

## Purpose

This folder demonstrates how a User Goal sits inside a Domain and owns Features.

## When To Use

Use this folder as a shape reference for new goals. A real goal should describe a stable user objective, not a solution.

## Expected Files

- `context.md`: goal context and traceability.
- `goal.md`: goal definition, users, outcomes, rules, and metrics.
- `journeys.md`: optional journey map when goal behavior spans multiple steps.
- `features/<feature>/`: solution features that serve the goal.

## Responsible Skill

Primary owner: User Goal AI.

## Next Step

Create or update a real goal, then identify candidate features that serve it.
