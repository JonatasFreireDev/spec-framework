# Example Feature

## Purpose

This folder demonstrates how a Feature sits under a User Goal and owns Use Cases.

## When To Use

Use this folder as a shape reference when creating real feature folders. A real feature should describe a concrete solution that supports the parent goal.

## Expected Files

- `context.md`: feature context and traceability.
- `feature.md`: feature definition, scope, rules, delivery, and risks.
- `decisions.md`: local feature decisions or links to global decisions.
- `use-cases/<use-case>/`: verified interactions that can feed specifications.

## Responsible Skill

Primary owner: Feature AI.

## Next Step

Create use cases only after the feature has enough scope and delivery priority to evaluate.
