# Example Domain

## Purpose

This folder is a reusable example of the canonical domain hierarchy. It should teach structure, naming, traceability, and expected artifact depth without representing real product scope.

## When To Use

Use this folder as a reference when creating a new domain, goal, feature, or use case. Do not treat example IDs, priorities, rules, or decisions as approved product commitments.

## Expected Files

- `context.md`: example domain context.
- `domain.md`: example domain definition.
- `decisions.md`: example decision notes.
- `goals/_example-goal/`: example goal hierarchy.

## Responsible Skill

Primary owner: Documentation Writer AI.

## Next Step

When bootstrapping a real domain, copy the pattern, not the content, and replace every `_example-*` placeholder with approved product terms.
