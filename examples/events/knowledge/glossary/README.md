# Glossary

## Purpose

Define domain language, product terms, and acronyms. Prefer one canonical term per concept.

## When To Use

Use this folder when terms repeat across artifacts or when ambiguity could cause inconsistent specifications, UI copy, analytics, or implementation.

## Expected Files

- `README.md`: folder purpose and usage.
- Future glossary files by domain or product area.

## Responsible Skill

Primary owner: Documentation Writer AI.

Supporting skills: Domain Architect AI and Specification AI.

## Next Step

When a term is approved, reference it from domain docs and specifications that depend on it.
