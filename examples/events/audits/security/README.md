# Security Audits

## Purpose

Store proactive and cross-cutting security audit artifacts, including threat modeling output and the living threat register.

## When To Use

Use this folder when security work spans more than one use case or when a threat should be tracked beyond a single Security Review.

## Expected Files

- `threat-register.md`: living register of threat scenarios, mitigations, owners, evidence, and residual risks.
- Future product-specific reports created by Threat Modeler AI or Audit Orchestrator.

## Responsible Skill

Primary owner: Threat Modeler AI.

Supporting skills: Security Review AI, QA AI, Audit Orchestrator, Product Historian AI.

## Next Step

Keep active threats linked from Security Review and from the relevant domain or use case context when they affect validation.
