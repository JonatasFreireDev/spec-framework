# Researches

## Purpose

Store research notes that support or challenge the product problem. Research can include market analysis, behavioral data, support tickets, usability studies, or operational observations.

## When To Use

Use this folder when evidence comes from sources other than a direct user interview. Keep raw evidence separate from interpretation so later agents can audit assumptions.

## Expected Files

- `YYYY-MM-DD-topic.md`: one research note per source or synthesis.
- Source links, date collected, method, findings, confidence, and implications.

## Responsible Skill

Primary owner: Problem Discovery AI.

## Next Step

Summarize validated findings in `../problem.md` and opportunity candidates in `../opportunities.md`.
