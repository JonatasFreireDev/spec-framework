# Skills v2 Orchestrators Index

## Purpose

This folder documents the orchestrators named in the framework. The executable orchestrator skills are stored in `.codex/skills/`.

## When To Use

Use orchestrators when work crosses multiple artifact types, requires sequencing, or must enforce approval gates. Do not use orchestrators to invent canonical product content owned by specialist skills.

## Expected Orchestrators

- Product Orchestrator: product foundation and domain setup.
- New Feature Orchestrator: feature to tasks flow.
- Audit Orchestrator: gap, conflict, dependency, impact, and consistency checks.
- Evolution Orchestrator: improvement candidate workflow.
- Documentation Orchestrator: context, indexes, templates, and decision synchronization.
- Release Orchestrator: release readiness and final gates.

## Responsible Skill

Primary owner: Documentation Orchestrator.
