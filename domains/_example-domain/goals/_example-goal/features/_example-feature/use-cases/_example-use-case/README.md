# Example Use Case

## Purpose

This folder demonstrates the executable artifact set for one use case.

## When To Use

Use this folder as a reference before creating a real use case. The real use case should contain enough detail to generate a Specification, Design, Implementation Plan, Execution Graph, Tasks, Tests, Analytics, and Audit.

## Expected Files

- `context.md`: use-case context and links.
- `use-case.md`: actor, goal, preconditions, flows, errors, and acceptance criteria.
- `specification.md`: implementation contract.
- `design.md`: UI/UX contract or explicit `Not applicable`.
- `implementation-plan.md`: technical plan without code.
- `execution-graph.json`: task dependency DAG.
- `tasks.md`: executable task list.
- `tests.md`: validation plan.
- `analytics.md`: measurement plan.
- `audit.md`: readiness or quality findings.

## Responsible Skill

Primary owner: Use Case AI until `use-case.md` is approved, then Specification AI and downstream planning skills.

## Next Step

For a real use case, complete `use-case.md`, run readiness checks, and only then create or approve the Specification.
